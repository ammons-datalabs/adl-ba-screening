How This Tool Works (show/hide details)

This tool checks flood risk using BCC open data. Results should be verified against FloodWise for critical decisions.

Data Sources (in priority order)

  • Implemented: BCC Parcel Metrics - Pre-computed flood flags from BCC for each lot/plan. Most accurate when lot-specific data is available.
  • Not yet implemented: Parcel Geometry Intersection - Would use actual parcel boundaries to check flood extent overlap. Planned for a future update.
  • Implemented: Point Buffer (30m) - Fallback when parcel data unavailable. Checks if a 30-metre radius around the geocoded address point intersects any flood extent.

Understanding the "Scope" Column

  • Lot-specific - Data is for the exact lot (e.g., Lot 4 on RP12345). Most reliable.
  • Plan-level (aggregated) - Lot-specific data wasn't available, so we're using the worst-case risk from the entire plan. This may overstate risk because the plan might contain multiple lots with different flood levels. Always verify these results in FloodWise.
  • Unknown - Typically from point buffer fallback where we only know if flood extents are nearby, not the specific risk level.

Understanding the Risk Column

  • High / Medium / Low / None - Lot-specific risk level from BCC data.
  • High* / Medium* / Low* - Risk marked with * is a plan-level estimate (see "Scope" above). Verify in FloodWise.
  • Check manually - Property is inside a flood extent but we couldn't determine the specific risk level. Verify in FloodWise.

Known Limitations

  • Plan-level results (*) may overstate risk - If we don't have lot-specific BCC metrics, we use plan-level data which shows the highest risk across all lots in that plan.
  • Point Buffer shows "Unknown" risk - We can detect flood extent proximity but not determine High/Medium/Low without parcel geometry intersection (coming soon).
  • Geocoding accuracy - Address matching depends on Google geocoding. Unusual addresses may geocode incorrectly.
